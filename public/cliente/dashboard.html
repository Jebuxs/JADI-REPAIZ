MODULO DASHBOASRD CLIENTE I. Estructura Visual (Lo que se ve) La interfaz presenta un diseño Minimalista Premium tipo "Dashboard de Accesos", optimizado para una experiencia móvil limpia sobre un fondo negro sólido. Cabecera de Bienvenida (Header): 1.-Texto de cortesía: "Bienvenido a JADI REPAIZ" en la esquina superior izquierda. 2.-Identidad dinámica: El nombre del usuario (ej. Jesus Eduardo Villa Orozco) resaltado en color Dorado. 3.-Avatar: Un icono circular de usuario en la parte derecha para dar sensación de perfil personal al estiulo boton. el cual nos dirije al perfil.js 4.-Listado de estilos de modulos que esta en estilo_visual.js y se puede escoger el estilo de modulos del listado II. Cuerpo Principal (Menú de Selección): Consta de dos grandes tarjetas centrales alineadas verticalmente con esquinas muy redondeadas (20px). 1.- boton 1: Tarjeta TIENDAS: 1.2.-Icono de una tienda de color segun el estilo del estilo_visual.js 1.3.-Título "TIENDAS" segun el estilo del estilo_visual.js. 1.4.-Descripción: "Ver locales y comprar productos". 2.-Boton 2: Tarjeta REPARACIONES: 2.1.-Icono de herramientas cruzadas en color segun el estilo del estilo_visual.js. 1.3.-Título "REPARACIONES" segun el estilo del estilo_visual.js. 1.3.-Descripción: "Solicitar nueva o ver seguimiento". III: Pie de Página: Un botón discreto y centrado de "Cerrar Sesión" con borde segun el estilo del estilo_visual.js fino, diseñado para no distraer de las funciones principales. IV. Estructura Lógica (Cómo funciona por dentro) El cerebro de esta página trabaja de la siguiente manera: VI. Gestión de Sesión (Seguridad): 1.-Local Storage: El sistema lee un objeto llamado session_jadi guardado en el navegador. 2.-Redirección Automática: Si el sistema no detecta este objeto (es decir, el usuario no se ha logueado), lo expulsa inmediatamente a la página de inicio (index.html). 3.-Se debe permitir el acceso a los usuarios registrados como administrador y cliente 'CLN', 'ADM' VII. Personalización Dinámica: La función init() se encarga de inyectar el nombre del cliente directamente desde los datos de la sesión al encabezado (#client-name), reemplazando el texto "Cargando...". VII. Sistema de Navegación: Cada tarjeta tiene un evento onclick. No son solo dibujos; funcionan como botones que disparan el cambio de ventana hacia 1.- Tiendas.html o 2.- Reparar.html. segun sea el caso VIII. Finalización de Sesión: La función cerrarSesion() realiza dos pasos críticos: 1.-Borra los datos de session_jadi (para que nadie más pueda entrar sin contraseña) 2.- Redirige al usuario al inicio. IX. Tecnologías Conectadas: Firebase: Aunque el código actual solo usa los datos de la sesión local, ya tiene importado el motor de Firebase Database, lo que significa que está listo para jalar datos en tiempo real de los locales o el estado de las reparaciones.
